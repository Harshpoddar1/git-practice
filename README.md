Git Practice Repo

A hands-on practice project built to learn Git and GitHub from the ground up — not just the commands, but the actual workflow used in real software teams.

What This Repo Demonstrates


Core Git workflow — staging, committing, viewing diffs and history
Branching & merging — creating feature branches and merging them back into main
Conflict resolution — deliberately created and manually resolved a merge conflict
Undoing changes — reset, revert, and checkout used correctly and safely
Stashing — saving work in progress without committing
Remote workflow — pushing, pulling, and cloning between local and GitHub
Pull Requests — full cycle of branch → PR → review → merge → cleanup
Tags & releases — marking a stable version (v1.0)
Repo hygiene — .gitignore to keep node_modules and .env out of version control
Recovery — using git reflog to recover a commit after a hard reset


Why I Built This

I wanted to actually understand what Git is doing internally — the staging area, how commits link together, what a merge conflict really looks like — instead of memorizing commands without knowing why they work. This repo is the record of that practice.
